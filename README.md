# wb_polki — трекер позиций в карусели «Рекомендуем» на Wildberries

Скрипт раз в сутки снимает позиции SKU (свои + конкуренты) в карусели  
«Рекомендуем» на страницах товаров WB и дописывает результат в Google Sheets.

---

## Структура файлов

```
wb_polki/
  polki_tracker.py   — главный скрипт
  config.py          — эндпоинт WB, категории, константы
  requirements.txt
  .env.example
  README.md
```

## 1. Верификация эндпоинта WB

> **Обязательный шаг перед первым запуском и после любых сбоев «пустой выдачи».**

WB периодически меняет адреса API. Актуальный эндпоинт нужно поймать в браузере:

1. Откройте любую карточку товара на [wildberries.ru](https://www.wildberries.ru).
2. DevTools → вкладка **Network** → фильтр `recom`.
3. Прокрутите страницу до блока «Рекомендуем» или дождитесь его загрузки.
4. Найдите XHR-запрос — вероятные хосты:
   - `recom.wb.ru`
   - `card.wb.ru`
5. Скопируйте полный URL и найдите параметры:
   - базовый путь → `WB_RECOM_URL` в `config.py`
   - значение параметра `dest` в вашей московской сессии → `DEST_MOSCOW` в `config.py`

**Зафиксированные значения (проверены на дату последнего коммита):**

| Параметр | Значение |
|---|---|
| `WB_RECOM_URL` | `https://recom.wb.ru/recom/client/v1/recomVendor` |
| `DEST_MOSCOW` | `-1257786` |

При изменении обновите `config.py` и этот файл.

## 2. Google Service Account

### 2.1 Создание сервисного аккаунта

1. [Google Cloud Console](https://console.cloud.google.com/) → ваш проект.
2. **APIs & Services → Library** → найдите **Google Sheets API** → Enable.
3. **IAM & Admin → Service Accounts** → Create Service Account.
4. Дайте имя, нажмите Continue → Done.
5. Откройте созданный аккаунт → **Keys → Add Key → JSON**.
6. Сохраните скачанный файл, например: `/home/deploy/wb_polki/sa-key.json`.
7. Запишите email аккаунта вида `xxx@yyy.iam.gserviceaccount.com`.

### 2.2 Расшаривание таблицы

Откройте Google Sheets → кнопка **Поделиться** → вставьте email сервисного аккаунта → **Редактор** → Готово.

## 3. Вкладка `Полки_вход` (заполняется вручную один раз)

Создайте вкладку `Полки_вход` в таблице со следующими колонками (первая строка — шапка):

| Категория | SKU | Бренд | Наш |
|---|---|---|---|
| Бинты | 328576305 | IRON BROTHERS | да |
| Бинты | 123456789 | Конкурент А | нет |
| … | … | … | … |

- **Категория** — одно из: Бинты, Тренажёр для шеи, Пояс атлетический, Шорты с тайтсами, Пояс карабин, Лямки, Наколенники.
- **SKU** — числовой nmId (только цифры, без пробелов).
- **Бренд** — название бренда.
- **Наш** — `да` для IRON BROTHERS, `нет` для конкурентов.

Полный список конкурентов — из мастер-блока таблицы (`Категория | SKU | ссылка на фото | ссылка на страницу | бренд`).

Наши SKU для заполнения:

| Категория | SKU |
|---|---|
| Бинты | 328576305 |
| Тренажёр для шеи | 387717955 |
| Пояс атлетический | 518527948 |
| Шорты с тайтсами | 943287734, 481633447 |
| Пояс карабин | 725637895 |
| Лямки | 378946759 |
| Наколенники | 310026736 |

## 4. Установка и настройка

```bash
# Клонировать / скопировать репозиторий на сервер
cd /home/deploy
git clone <repo-url> wb_polki
cd wb_polki

# Создать виртуальное окружение
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Настроить .env
cp .env.example .env
nano .env          # заполнить SHEET_ID, GOOGLE_SA_JSON, TELEGRAM_TOKEN, TELEGRAM_CHAT_ID
```

## 5. Тестовый запуск

```bash
source venv/bin/activate
python polki_tracker.py
```

Убедитесь, что во вкладке `Полки_лог` появились строки с сегодняшней датой.

## 6. Деплой на relaxdev.ru (Docker)

relaxdev.ru запускает проекты как Docker-контейнеры. Системный cron не нужен —
расписание встроено в `scheduler.py`, контейнер работает постоянно.

### 6.1 Подготовка JSON-ключа сервисного аккаунта

Содержимое JSON-файла нужно передать в контейнер. Самый простой способ —
через переменную окружения `GOOGLE_SA_JSON_CONTENT` (см. ниже).

Добавьте в `polki_tracker.py` поддержку этого варианта (одна строка в `get_gsheet`):

```python
# В функции get_gsheet(), перед Credentials.from_service_account_file:
sa_content = os.getenv("GOOGLE_SA_JSON_CONTENT")
if sa_content:
    import json, tempfile
    tmp = tempfile.NamedTemporaryFile(delete=False, suffix=".json", mode="w")
    tmp.write(sa_content)
    tmp.close()
    sa_path = tmp.name
```

### 6.2 Переменные окружения в relaxdev.ru

В настройках проекта → **Environment Variables** добавьте:

| Переменная | Значение |
|---|---|
| `SHEET_ID` | `1Dz0s7VDTtbC90mXk97-R19bseQYnSPI7d3I3FVgDf0Y` |
| `GOOGLE_SA_JSON_CONTENT` | Весь JSON-ключ одной строкой (скопируйте содержимое файла) |
| `TELEGRAM_TOKEN` | Токен вашего бота |
| `TELEGRAM_CHAT_ID` | ID чата для алертов |

### 6.3 Деплой

1. Загрузите репозиторий на GitHub.
2. relaxdev.ru → **Импорт и деплой** → выберите репозиторий.
3. Платформа найдёт `Dockerfile` и соберёт образ автоматически.
4. Контейнер запустится и будет ждать 02:00 МСК для первого прогона.

Логи смотреть: **VM → finboad logs** (аналогично для вашего контейнера) или через Live Terminal.

## 7. Схема вкладки `Полки_лог`

| Колонка | Описание |
|---|---|
| Дата | YYYY-MM-DD |
| Время | HH:MM:SS (МСК) |
| Регион | Москва |
| Тип_полки | Рекомендуем |
| Категория | Бинты / … |
| Базовый_SKU | nmId карточки, для которой запрашивали карусель |
| Базовый_бренд | Бренд базовой карточки |
| Базовый_наш | да / нет |
| Найденный_SKU | nmId искомой карточки |
| Найденный_бренд | Бренд найденной карточки |
| Найденный_наш | да / нет |
| Позиция | 1–100 или >100 |
| Длина_выдачи | Фактическое число позиций в карусели (если < 100, WB вернул короткий список) |

## 8. Telegram-алерты

Скрипт шлёт уведомление при:
- пустом watchlist (вкладка `Полки_вход` не заполнена),
- нуле собранных строк (WB API недоступен),
- любом необработанном исключении.

Используйте существующего бота на relaxdev.ru — впишите его `TELEGRAM_TOKEN` и нужный `TELEGRAM_CHAT_ID` в `.env`.

---

## 9. Контроль качества

Раз в неделю позиции сверяются вручную через **Evirma** (вне скрипта).  
Если `Длина_выдачи` < 100 — WB вернул короткий список; не считать позицию ">100" реальным провалом.
